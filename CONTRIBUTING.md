# Contributing to XLog GUI

Thank you for your interest in contributing to XLog GUI! We welcome contributions from everyone.

## How to Contribute

### Reporting Bugs
1. Check the issue tracker to ensure the bug hasn't already been reported.
2. If the issue is new, open a new issue with a clear title and detailed description.
3. Include steps to reproduce, expected behavior, and screenshots if applicable.

### Suggesting Enhancements
1. Open a new issue with the "enhancement" label.
2. Clearly describe the feature you would like to see and why it would be useful.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or fix (`git checkout -b feature/amazing-feature`).
3. Commit your changes (`git commit -m 'Add some amazing feature'`).
4. Push to the branch (`git push origin feature/amazing-feature`).
5. Open a Pull Request.

## Code Style
- Follow standard Kotlin coding conventions.
- Use `rustfmt` for Rust code.
- Ensure all tests pass before submitting.
